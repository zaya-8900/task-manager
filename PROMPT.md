# Task Manager Development

## Your Mission
Build the task manager described in SPEC.md.

## Each Iteration

1. Read SPEC.md to understand the goal
2. Check IMPLEMENTATION_PLAN.md for the next uncompleted task
3. Implement ONLY that one task
4. Run tests: `pytest tests/ -v`
5. If tests pass:
   - Mark task complete in IMPLEMENTATION_PLAN.md
   - Commit: "feat: [description]"
6. If tests fail:
   - Fix the failing tests
   - Do NOT move on until tests pass
7. Exit when done (loop will restart)

## Constraints

- One task per iteration
- Tests must pass before committing
- Keep changes focused
- If stuck for 2+ iterations on the same task, leave a note and move on

## Quality Standards

- Use type hints
- Handle errors gracefully
- Write docstrings for functions
- Keep functions small and focused
